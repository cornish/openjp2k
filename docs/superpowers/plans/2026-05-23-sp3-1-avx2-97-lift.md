# SP3.1 — AVX2 9/7 Float Lift Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an AVX2 path (`__m256`, 8 floats/op) for the 9/7 float IDWT lift kernels, gated by compile-time `#ifdef __AVX2__`. Targets the ~16% of decode wall-time the D2 profile attributed to `opj_dwt_decode_real` on lossy archival workloads.

**Architecture:** New `opj_v8dwt_decode_step1_avx2` and `opj_v8dwt_decode_step2_avx2` mirror the existing `_sse` kernels structurally — same loop shape, same operation sequence (mul + add, no FMA), just `__m256` instead of `__m128`. Selection happens at `opj_v8dwt_decode` via an `#ifdef __AVX2__` / `#elif defined(__SSE__)` / `#elif defined(__ARM_NEON)` ladder. The data layout (`opj_v8_t` = 8 floats = 32 bytes) is unchanged.

**Tech Stack:** C99, x86 AVX2 intrinsics (`<immintrin.h>`, already pulled in via `<x86intrin.h>` at dwt.c:50 when `__AVX2__` is defined). `-march=native` build (bench host has AVX2).

- [ ] **Step 2: Add the AVX2 kernels above the SSE2 ones.**

Locate the existing `#ifdef __SSE__` block at dwt.c:3243. Just above it (around line 3242), insert:

```c
#ifdef __AVX2__

/* SP3.1: AVX2 variant of opj_v8dwt_decode_step1_sse.
 *
 * One opj_v8_t (8 floats, 32 bytes) is one __m256. The SSE2 path
 * processes the same record as two consecutive __m128 ops; AVX2 does
 * it in one. The vw += 2 advance skips the alternate (low/high) band
 * member that step2 will touch in a separate pass. No FMA — keep the
 * mul + add sequence byte-identical to the SSE2 path. */
static void opj_v8dwt_decode_step1_avx2(opj_v8_t* w,
                                        OPJ_UINT32 start,
                                        OPJ_UINT32 end,
                                        const __m256 c)
{
    __m256* OPJ_RESTRICT vw = (__m256*) w;
    OPJ_UINT32 i = start;
    /* 2 = NB_ELTS_V8 / 8 floats per __m256 * 2 band slots per stride */
    vw += 2 * start;
    for (; i < end; ++i, vw += 2) {
        vw[0] = _mm256_mul_ps(vw[0], c);
    }
}

/* SP3.1: AVX2 variant of opj_v8dwt_decode_step2_sse. */
static void opj_v8dwt_decode_step2_avx2(opj_v8_t* l, opj_v8_t* w,
                                        OPJ_UINT32 start,
                                        OPJ_UINT32 end,
                                        OPJ_UINT32 m,
                                        __m256 c)
{
    __m256* OPJ_RESTRICT vl = (__m256*) l;
    __m256* OPJ_RESTRICT vw = (__m256*) w;
    OPJ_UINT32 i;
    OPJ_UINT32 imax = opj_uint_min(end, m);
    if (start == 0) {
        if (imax >= 1) {
            vw[-1] = _mm256_add_ps(vw[-1],
                         _mm256_mul_ps(_mm256_add_ps(vl[0], vw[0]), c));
            vw += 2;
            start = 1;
        }
    } else {
        vw += start * 2;
    }

    i = start;
    for (; i < imax; ++i) {
        vw[-1] = _mm256_add_ps(vw[-1],
                     _mm256_mul_ps(_mm256_add_ps(vw[-2], vw[0]), c));
        vw += 2;
    }
    if (m < end) {
        assert(m + 1 == end);
        c = _mm256_add_ps(c, c);
        vw[-1] = _mm256_add_ps(vw[-1], _mm256_mul_ps(c, vw[-2]));
    }
}

#elif defined(__SSE__)
```

The `#elif defined(__SSE__)` replaces the existing `#ifdef __SSE__` at the start of the SSE block. This means: when AVX2 is defined, only the AVX2 kernels compile. When AVX2 is not defined but SSE is, the SSE kernels compile. When neither, NEON (or scalar fallback) takes over.

**Note: NB_ELTS_V8 is currently 8 (dwt.c:96).** The kernels assume this; if it ever changes, both AVX2 and SSE2 kernels need re-derivation. The existing `_sse` kernel has a `/* To be adapted if NB_ELTS_V8 changes */` comment for the same reason.
