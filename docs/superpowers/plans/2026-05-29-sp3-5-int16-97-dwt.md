# SP-3.5 — Int16 9/7 DWT Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an AVX2 int16 Q-format path for the 9/7 lossy inverse DWT, dispatched per-tilec when source precision ≤ 8 bits AND `whole_tile_decoding` is true. Targets 8-bit lossy decode (WSI 1024² tiles, photographic, 8-bit DICOM) with a predicted +5-8% wall-clock gain on those workloads. Closes most of the remaining grok lead on the 1024² WSI tile workload.

**Architecture:** A new orchestrator `opj_dwt_decode_tile_real_int16` parallels the existing float `opj_dwt_decode_tile_real`. It packs `tilec->data` (int32) into a Q5.11 int16 scratch buffer, runs the inverse 9/7 lift in int16 across all 5 decomposition levels (with mirrored horizontal and vertical kernels using `_mm256_mulhrs_epi16` for hardware-rounded Q-format multiplies), then unpacks back to int32 with rounding shift. Lift constants use Q1.15 for |c| < 1 (β, γ, δ, 1/K) and Q2.14 for α and K (which exceed Q1.15 range). `opj_dwt_decode_real` chooses the orchestrator based on `image_comp.prec`, `whole_tile_decoding`, and `__AVX2__` availability. The float path is untouched.

**Tech Stack:** C99, x86 AVX2 intrinsics (`__m256i` carrying 16 int16 lanes), `_mm256_mulhrs_epi16` for Q1.15 rounding multiply, `OPJ_ENABLE_AVX2` cmake gate (from SP3.1).

- `_mm256_sub_ps` → `_mm256_subs_epi16` (saturating)

- Loop stride: 8 → 16 (16 lanes per vector instead of 8)

### Step 2: Add Q5.11 fixed-point macros and Q1.15 multiply-add helpers near the existing SP3.1 macros

Locate the existing SIMD-macro block in dwt.c (where `__AVX2__` AVX2 macros for the float lift live, similar to SP3.2's `VREG_S16` block). Add immediately below it:

```c
#ifdef __AVX2__
/* SP-3.5: int16 Q5.11 mirror macros for the 9/7 IDWT.  16 int16 lanes
 * per __m256i (vs 8 float lanes in VREG).  Q5.11 input/output format
 * (5 integer bits, 11 fractional bits) covers 8-bit precision input
 * with margin through 5 decomposition levels. */
#define VREG_S16        __m256i
#define LOADU_S16(x)    _mm256_loadu_si256((const VREG_S16*)(x))
#define STOREU_S16(x,v) _mm256_storeu_si256((VREG_S16*)(x), v)
#define ADD_S16(a,b)    _mm256_adds_epi16(a, b)   /* saturating */
#define SUB_S16(a,b)    _mm256_subs_epi16(a, b)
#define SET1_S16(x)     _mm256_set1_epi16((int16_t)(x))

/* Q1.15 rounding multiply: result = (a * b + 0x4000) >> 15
 * For coefficients in [-1, 1).  Used for beta, gamma, delta, 1/K. */
#define MULHRS_Q15(a,b) _mm256_mulhrs_epi16(a, b)

/* Q2.14 rounding multiply for coefficients in [-2, 2): alpha, K.
 * Equivalent to Q1.15 multiply then left-shift by 1 (saturating). */
#define MULHRS_Q14(a,b) _mm256_adds_epi16(_mm256_mulhrs_epi16(a,b), \
                                          _mm256_mulhrs_epi16(a,b))

/* Q1.15 constants (×32768): */
#define Q15_BETA        ((int16_t)(-1736))    /* -0.052980 */
#define Q15_GAMMA       ((int16_t)(28932))    /* +0.882911 */
#define Q15_DELTA       ((int16_t)(14533))    /* +0.443507 */
#define Q15_INVK        ((int16_t)(26637))    /* +0.812893 */

/* Q2.14 constants (×16384): */
#define Q14_ALPHA       ((int16_t)(-25996))   /* -1.586134 */
#define Q14_K           ((int16_t)(20157))    /* +1.230174 */
#endif /* __AVX2__ */
```

(Coefficient values may need ±1 LSB tuning during implementation — round-to-nearest in the Q-format conversion. Recompute in Python and validate before locking.)

### Step 3: Add the pack/unpack helpers

Q5.11 pack: input is int32 (dequantized 9/7 coefficient, magnitude ≤ ±32). Output is int16 with the value shifted left by 11 and saturated.

```c
#ifdef __AVX2__
/* SP-3.5: Pack int32 -> int16 Q5.11 with saturation.
 * scratch: output int16 array, n elements.
 * src: input int32 array, n elements (values in [-32, +32) for 8-bit input).
 * Performs (int32_value << 11) clamped to [INT16_MIN, INT16_MAX].
 * Equivalent scalar:
 *   for (i = 0; i < n; ++i) {
 *     int32_t shifted = src[i] << 11;
 *     if (shifted > INT16_MAX) scratch[i] = INT16_MAX;
 *     else if (shifted < INT16_MIN) scratch[i] = INT16_MIN;
 *     else scratch[i] = (int16_t)shifted;
 *   }
 * Vectorized: load 8 int32, shift left 11 (clamping needed since shift
 * can overflow int32 for values > 2^20 — though we expect values < 2^5,
 * the SIMD path uses _mm256_packs_epi32 which saturates).
 */
static void opj_dwt_pack_int32_to_int16_q511(int16_t* dst,
                                              const OPJ_INT32* src,
                                              OPJ_SIZE_T n);

/* SP-3.5: Unpack int16 Q5.11 -> int32 with rounding.
 * dst: output int32, n elements.  src: input int16 Q5.11.
 * Performs (src[i] + 1024) >> 11 with sign extension.
 * Vectorized: load 16 int16, sign-extend low 8 and high 8 via
 * _mm256_cvtepi16_epi32, add 1024 (the rounding offset = 2^10),
 * arithmetic-shift-right by 11, store.
 */
static void opj_dwt_unpack_int16_q511_to_int32(OPJ_INT32* dst,
                                                const int16_t* src,
                                                OPJ_SIZE_T n);
#endif
```

Implementation (~50 LOC each, AVX2 inner loop with scalar tail for the last < 8 / < 16 elements).

### Step 4: Add the 4 int16 lift kernels

Each kernel is a structural mirror of the corresponding SP3.1 float kernel. Use the SP3.2 5/3 kernels as the template for the int16 lane-count loop structure (16 lanes per `__m256i`), the SP3.1 9/7 float kernels as the template for the lift formula structure (4 lift steps + 2 scale steps).

The lift step replacement: where SP3.1 float code does

```c
/* SP3.1 float kernel inner loop */
v_acc = _mm256_fmadd_ps(coeff, _mm256_add_ps(d_minus1, d_plus0), s);
```

the SP3.5 int16 equivalent is

```c
/* SP3.5 int16 kernel inner loop (for Q1.15 coefficients: beta, gamma, delta, 1/K) */
v_sum_d = ADD_S16(d_minus1, d_plus0);
v_mult  = MULHRS_Q15(v_sum_d, SET1_S16(Q15_BETA));   /* or GAMMA/DELTA */
v_acc   = ADD_S16(s, v_mult);

/* For Q2.14 coefficients (alpha, K), use MULHRS_Q14 instead. */
```

Note that `_mm256_mulhrs_epi16` rounds to nearest (using the +0x4000 offset) so this exactly matches the documented Q1.15 semantics. There is no fused multiply-add for int16 in AVX2 — multiplication and addition are separate instructions but each is one cycle.

Write the 4 kernels (`opj_v16dwt_decode_step1_int16_avx2`, `opj_v16dwt_decode_step2_int16_avx2`, `opj_v16dwt_interleave_h_int16_avx2`, `opj_v16dwt_interleave_v_int16_avx2`) line-for-line from the SP3.1 float versions.

### Step 7: Add the orchestrator

```c
#ifdef __AVX2__
static OPJ_BOOL opj_dwt_decode_tile_real_int16(opj_thread_pool_t* tp,
                                                opj_tcd_tilecomp_t* tilec,
                                                OPJ_UINT32 numres)
{
    /* Body modeled on opj_dwt_decode_tile_real (the existing SP3.1 float
     * orchestrator).  Substitutions:
     *   - allocate int16 scratch via opj_aligned_32_malloc
     *   - pack tilec->data into scratch via opj_dwt_pack_int32_to_int16_q511
     *   - per-resolution loop:
     *       - for each row: opj_dwt97_decode_h_func_int16
     *       - for each col-stripe: opj_dwt97_decode_v_func_int16 (via job submit)
     *   - unpack scratch back to tilec->data via opj_dwt_unpack_int16_q511_to_int32
     *   - free scratch
     * Thread pool jobs are submitted via opj_thread_pool_submit_job for the
     * vertical pass, mirroring the float orchestrator's job submission.
     * Horizontal pass runs synchronously (mirrors float orchestrator).
     */
}
#endif
```

The scratch buffer is allocated per tilec (no pool integration in SP-3.5; see spec's Open Question #2). The body is ~150 LOC structurally similar to the float orchestrator.

### Step 8: Add the dispatch hook in `opj_dwt_decode_real`

Locate `opj_dwt_decode_real` (it's near the bottom of dwt.c, the public entry for the 9/7 IDWT path):

```c
OPJ_BOOL opj_dwt_decode_real(opj_tcd_t *p_tcd, opj_tcd_tilecomp_t* tilec,
                             OPJ_UINT32 numres)
{
    if (p_tcd->whole_tile_decoding) {
#ifdef __AVX2__
        /* SP-3.5: int16 Q5.11 path for <=8-bit precision components. */
        {
            const OPJ_UINT32 compno = (OPJ_UINT32)(tilec -
                                      p_tcd->tcd_image->tiles->comps);
            const opj_image_comp_t *image_comp = &p_tcd->image->comps[compno];
            if (image_comp->prec <= 8) {
                return opj_dwt_decode_tile_real_int16(p_tcd->thread_pool, tilec, numres);
            }
        }
#endif
        return opj_dwt_decode_tile_real(p_tcd->thread_pool, tilec, numres);
    } else {
        return opj_dwt_decode_partial_tile_real(tilec, numres);
    }
}
```

Note the partial branch is untouched (per scope boundary in the spec).
